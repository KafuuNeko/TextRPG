## 1. 核心概念：一切皆配置

这个引擎的设计哲学是：**游戏逻辑写在代码里，游戏内容写在配置里。**

你要做的事情就像填表格——在配置文件里填入你想要的属性、技能、敌人、地图……引擎会自动把它们组装成一个完整的游戏。

**打个比方：** 引擎是一台自动贩卖机，配置文件是你往里面放的商品。你换一批商品，贩卖机就变成了不同的店。

---

## 2. 配置文件在哪里

所有配置文件都在这个目录下：

```
src/main/resources/config/
```

| 文件 | 控制什么 |
|------|---------|
| `attributes.yaml` | 角色有哪些属性（力量、HP、攻击力……） |
| `skills.yaml` | 有哪些技能（火球术、治疗术……） |
| `buffs.yaml` | 有哪些状态效果（灼烧、中毒、护盾……） |
| `combat.yaml` | 敌人数据 + 战斗伤害公式 |
| `nodes.yaml` | 地图有哪些区域、怎么连接 |
| `npcs.yaml` | NPC 是谁、在哪、能做什么 |
| `quests.yaml` | 任务内容和奖励 |
| `inventory.yaml` | 背包容量设置 |
| `commands.yaml` | 玩家可以输入哪些指令 |
| `app.yaml` | 连接设置（QQ机器人、AI模型） |
| `ai.yaml` | AI 对话的行为规则 |

改完配置后**重启游戏**就会生效（运行 `gradlew run`）。

## 3. YAML 格式速成

配置文件用的是 YAML 格式，非常接近自然语言。3 分钟就能学会：

### 基本规则

```yaml
# 这是注释，不会生效

名字: "值"           # 冒号后面有个空格！
数字: 42             # 数字不用引号
开关: true           # true 或 false

# 缩进表示层级关系（用空格，不要用 Tab！）
角色:
  名字: "勇者"
  等级: 1

# 列表用 - 开头
技能列表:
  - "火球术"
  - "治疗术"
```

### 最容易犯的错

| 错误 | 正确 |
|------|------|
| `名字:"值"` （冒号后没空格） | `名字: "值"` |
| 用 Tab 缩进 | 用 2 个空格缩进 |
| 中文引号 `"值"` | 英文引号 `"值"` |

## 4. 修改属性体系

**文件：** `attributes.yaml`

属性分两种：

- **一级属性**：直接存储的数值（力量、体质、等级……）
- **二级属性**：由公式从其他属性计算出来的（攻击力、生命上限……）

### 修改现有属性

比如想把初始力量改成 15：

```yaml
  strength:
    tier: 1
    displayName: "力量"
    defaultValue: 15    # 改这个数字
```

### 添加新的一级属性

在 `attributes:` 下面加一段：

```yaml
  luck:
    tier: 1
    displayName: "幸运"
    description: "影响暴击和掉落"
    defaultValue: 5
    min: 0
    max: 100
```

### 添加新的二级属性

二级属性靠公式计算，公式可以引用其他属性：

```yaml
  crit_rate:
    tier: 2
    displayName: "暴击率"
    formula: "agility * 0.5 + luck * 0.3"    # 引用敏捷和幸运
    min: 0
    max: 100
```

### 公式语法

| 写法 | 含义 |
|------|------|
| `strength * 2` | 力量 x 2 |
| `constitution * 10 + 50` | 体质 x 10 + 50 |
| `agility * 0.5 + luck * 0.3` | 可以引用多个属性 |
| `(strength + agility) * 1.5` | 支持括号 |

## 5. 添加 / 修改技能

**文件：** `skills.yaml`

### 一个技能长什么样

```yaml
skills:
  fireball:                        # 技能的唯一标识（英文，不能重复）
    displayName: "火球术"            # 玩家看到的名字
    description: "向敌人投掷火球"     # 技能描述
    cost:
      current_mp: 25               # 消耗 25 点 MP
    cooldown: 2                    # 使用后冷却 2 回合
    effects:                       # 技能效果（按顺序执行）
      - type: "modify_attribute"   # 效果类型：修改属性
        target: "target"           # 作用目标：敌人
        attribute: "current_hp"    # 修改哪个属性：当前HP
        amount: "-(magic_attack * 1.5 + 30)"   # 伤害公式（负数=扣血）
```

### 添加一个新技能

在 `skills:` 下面加一段：

```yaml
  ice_bolt:
    displayName: "冰箭术"
    description: "发射一支冰箭"
    cost:
      current_mp: 15
    cooldown: 1
    effects:
      - type: "modify_attribute"
        target: "target"
        attribute: "current_hp"
        amount: "-(magic_attack * 1.2)"
      - type: "add_buff"
        target: "target"
        buffId: "frozen"           # 附加一个冰冻 Buff（需要在 buffs.yaml 里定义）
        duration: 2
```

### 效果类型速查

| type | 用途 | 常用参数 |
|------|------|---------|
| `modify_attribute` | 加减属性值（伤害/治疗） | target, attribute, amount |
| `add_buff` | 给目标上 Buff | target, buffId, duration |
| `remove_buff` | 移除 Buff | target, buffId |
| `send_message` | 显示一段文字 | target, message |

### 目标选择器

| target 值 | 含义 |
|-----------|------|
| `self` | 自己 |
| `target` | 当前攻击目标 |
| `all_enemies` | 所有敌人 |
| `all_allies` | 所有队友 |

### 伤害公式怎么写

- **负数 = 扣血**：`"-(physical_attack * 2)"` 表示造成物攻 x2 的伤害
- **正数 = 加血**：`"intelligence * 2 + 50"` 表示恢复 智力x2+50 的生命

公式里可以用任何在 `attributes.yaml` 里定义过的属性名。

## 6. 添加 / 修改 Buff 和状态异常

**文件：** `buffs.yaml`

### 一个 Buff 长什么样

```yaml
buffs:
  burn_dot:                         # Buff 唯一标识
    displayName: "灼烧"
    description: "每回合受到火焰伤害"
    maxStacks: 3                    # 最多叠 3 层
    duration: 3                     # 持续 3 回合
    stackPolicy: "stack"            # 叠加规则（见下表）
    modifiers:                      # 持续期间的属性修改（穿上/脱下效果）
      - attribute: "defense"
        type: "flat"
        value: -5                   # 降低 5 点防御
    tickEffects:                    # 每回合触发的效果
      - type: "modify_attribute"
        target: "self"
        attribute: "current_hp"
        amount: "-8 * stacks"       # 每层每回合扣 8 血
```

### 叠加规则

| stackPolicy | 效果 |
|-------------|------|
| `refresh` | 重复施加时只刷新持续时间，不叠加 |
| `stack` | 重复施加时层数增加，效果增强 |
| `independent` | 每次施加独立存在，各自倒计时 |

### 添加新 Buff 示例

```yaml
  frozen:
    displayName: "冰冻"
    maxStacks: 1
    duration: 2
    stackPolicy: "refresh"
    modifiers:
      - attribute: "agility"
        type: "flat"
        value: -10                 # 降低敏捷
```

## 7. 添加 / 修改敌人

**文件：** `combat.yaml`，在 `enemies:` 下面

### 一个敌人长什么样

```yaml
enemies:
  slime:                            # 敌人唯一标识
    displayName: "史莱姆"             # 玩家看到的名字
    attributes:                     # 敌人的属性值
      current_hp: 30
      max_hp: 30
      physical_attack: 5
      defense: 2
    skills: ["basic_attack"]        # 敌人会用的技能（引用 skills.yaml 的 key）
    aiRules:                        # AI 行为规则（按优先级排列）
      - condition: "1"              # 条件：1 = 始终成立
        action: "basic_attack"      # 执行普通攻击
        priority: 1                 # 优先级（数字越大越优先）
    rewards:                        # 击败后的奖励
      expFormula: "15"              # 获得 15 经验
      drops:                        # 掉落物
        - item: "8"                 # 物品模板 ID（见物品章节）
          chance: 0.6               # 60% 概率掉落
          quantity: "1"             # 掉落 1 个
```

### 添加新敌人

```yaml
  skeleton:
    displayName: "骷髅兵"
    attributes:
      current_hp: 80
      max_hp: 80
      physical_attack: 18
      defense: 8
    skills: ["basic_attack", "power_strike"]
    aiRules:
      - condition: "self.current_hp < self.max_hp * 0.5"   # 血量低于 50% 时
        action: "power_strike"                               # 用强力一击
        priority: 5
      - condition: "1"                                       # 默认行为
        action: "basic_attack"
        priority: 1
    rewards:
      expFormula: "40"
      drops:
        - item: "9"
          chance: 0.3
          quantity: "1"
```

### AI 条件怎么写

| 条件写法 | 含义 |
|---------|------|
| `"1"` | 始终成立（用作默认行为） |
| `"self.current_hp < self.max_hp * 0.3"` | 自己血量低于 30% |
| `"self.current_mp > 14"` | 自己 MP 大于 14 |

`self.xxx` 引用敌人自己的属性，`target.xxx` 引用玩家的属性。

**注意：** 优先级（priority）大的规则先检查。引擎从上往下找，第一个条件满足的就执行。所以把"血少时治疗"之类的特殊规则放在前面（高优先级），"默认普攻"放在最后（低优先级）。

### 让敌人出现在地图上

光定义敌人还不够，需要在 `nodes.yaml` 的对应节点里加上：

```yaml
  forest_path:
    entities:
      enemies: ["slime", "skeleton"]    # 加上新敌人的 key
```

## 8. 修改地图

**文件：** `nodes.yaml`

### 地图是一张有向图

每个区域是一个"节点"，节点之间通过"连接"相连。玩家只能沿着连接移动。

```
[村庄广场] ──→ [铁匠铺]
     │
     ↓
  [村口] ──→ [森林小径]
```

### 一个节点长什么样

```yaml
nodes:
  forest_path:                      # 节点唯一标识（英文）
    displayName: "森林小径"           # 玩家看到的名字
    description: "蜿蜒的小径，偶尔能看到史莱姆在草丛中蹦跳。"
    tags: ["combat_zone"]           # 标签（见下表）
    connections:                    # 可以从这里去哪
      - target: "village_gate"      # 目标节点的 key
        display: "→ 村口"            # 玩家看到的方向描述
      - target: "deep_forest"
        display: "→ 深林"
      - target: "bandit_camp"
        display: "→ 盗贼营地"
        requires:                   # 需要满足条件才能通行
          minAttribute:
            level: 3                # 需要 3 级
    entities:
      enemies: ["slime"]            # 这里有哪些敌人
      npcs: []                      # 这里有哪些 NPC
```

### 标签（tags）

| 标签 | 含义 |
|------|------|
| `safe_zone` | 安全区域（无战斗） |
| `combat_zone` | 战斗区域（有敌人） |
| `shop` | 有商店 |
| `gatherable` | 可采集 |
| `boundary` | 边界节点（可用 /探索 生成新区域） |

### 添加新区域

```yaml
  crystal_cave:
    displayName: "水晶洞窟"
    description: "洞壁上镶嵌着发光的水晶，空气中弥漫着魔力的气息。"
    tags: ["combat_zone"]
    connections:
      - target: "deep_forest"       # 连回深林
        display: "→ 深林"
    entities:
      enemies: ["skeleton"]         # 放上新敌人
```

**别忘了：** 同时在 `deep_forest` 的 connections 里加一条通往 `crystal_cave` 的连接，否则玩家走不过去！

```yaml
  deep_forest:
    connections:
      # ... 已有的连接 ...
      - target: "crystal_cave"
        display: "→ 水晶洞窟"
```

## 9. 添加 / 修改 NPC

**文件：** `npcs.yaml`

```yaml
npcs:
  blacksmith:                       # NPC 唯一标识
    displayName: "铁匠"
    description: "身材魁梧的铁匠，手艺在方圆百里首屈一指。"
    location: "blacksmith"          # 所在的地图节点 key
    prompt: "你是一个寡言少语但技艺精湛的铁匠。"   # AI 对话的角色设定
    functions:                      # NPC 提供的功能
      - type: "shop"               # 商店
        items: "2,5"               # 卖的物品（模板 ID，逗号分隔）
```

### NPC 功能类型

| type | 含义 | 额外参数 |
|------|------|---------|
| `shop` | 商店 | `items: "模板ID,模板ID"` |
| `quest_giver` | 任务发放 | `quest_ids: "任务key"` |
| `dialogue` | AI 对话 | 无（用 prompt 控制） |

### 添加新 NPC

```yaml
  guard:
    displayName: "门卫"
    description: "守在村口的年轻卫兵。"
    location: "village_gate"
    prompt: "你是一个认真负责的年轻卫兵，会提醒冒险者注意安全。"
    functions:
      - type: "dialogue"
```

## 10. 添加 / 修改任务

**文件：** `quests.yaml`

```yaml
quests:
  herb_quest:                       # 任务唯一标识
    displayName: "采药委托"
    description: "药师需要你前往药草田采集草药。"
    giver: "herbalist"              # 任务发放者（NPC 的 key）
    objectives:                     # 目标列表（全部完成才算完成）
      - type: "reach_node"          # 目标类型：到达某节点
        node: "herb_field"          # 目标节点
        quantity: 1                 # 到达 1 次
    rewards:                        # 奖励
      exp: 50                       # 经验
      items:
        - item: "6"                 # 物品模板 ID
          quantity: 3               # 数量
      attributeChange:
        gold: 100                   # 金币 +100
    requires:                       # 接取条件
      registered: true
```

### 目标类型

| type | 含义 | 关键参数 |
|------|------|---------|
| `reach_node` | 到达指定地图节点 | `node: "节点key"` |
| `kill_enemy` | 击杀指定敌人 | `enemy: "敌人key"`, `quantity: 数量` |
| `collect_item` | 收集指定物品 | `item: "物品ID"`, `quantity: 数量` |
| `talk_to_npc` | 与指定 NPC 对话 | `npc: "NPC的key"` |

### 添加新任务

```yaml
  patrol_quest:
    displayName: "巡逻任务"
    description: "门卫请求你巡查森林小径的安全状况。"
    giver: "guard"
    objectives:
      - type: "reach_node"
        node: "forest_path"
        quantity: 1
    rewards:
      exp: 30
      attributeChange:
        gold: 50
```

别忘了在对应 NPC 的 functions 里加上 `quest_giver`：

```yaml
  guard:
    functions:
      - type: "quest_giver"
        quest_ids: "patrol_quest"
```

```kotlin
ItemTemplate(
    name = "魔法杖",
    type = ItemType.EQUIPMENT,
    subType = ItemSubType.WEAPON,
    rarity = ItemRarity.BLUE,      // WHITE=白 GREEN=绿 BLUE=蓝 PURPLE=紫 ORANGE=橙
    stackable = false,
    baseStats = """{"magic_attack": 20, "intelligence": 5}""",
    levelReq = 5,                  // 需要 5 级才能装备
    price = 300,
    description = "蕴含魔力的法杖。"

```

```kotlin
ItemTemplate(
    name = "解毒草",
    type = ItemType.CONSUMABLE,
    stackable = true,
    price = 15,
    description = "可以解除中毒状态。"

```

## 12. 修改战斗公式

**文件：** `combat.yaml` 的 `combat:` 部分

```yaml
combat:
  damageFormula: "attacker.physical_attack - defender.defense * 0.5"
  critCheck: "random(100) - attacker.crit_rate"  # 结果 <= 0 时暴击
  critMultiplier: 1.5       # 暴击伤害倍率
  minDamage: 1              # 最低伤害（不会打出 0）
  defaultTimeoutSeconds: 60 # 玩家操作超时（秒）
```

### 公式中可用的前缀

| 前缀 | 含义 |
|------|------|
| `attacker.xxx` | 攻击方的属性 |
| `defender.xxx` | 防御方的属性 |
| `random(100)` | 0~99 的随机数 |

### 示例修改

想让伤害更高：
```yaml
  damageFormula: "attacker.physical_attack * 1.5 - defender.defense * 0.3"
```

想取消暴击系统：
```yaml
  critCheck: "1"   # 永远 > 0，即永远不暴击
```

## 13. 修改商店和经济

### 修改商品列表

在 `npcs.yaml` 中，修改 NPC 的 shop 功能里的 `items`：

```yaml
  blacksmith:
    functions:
      - type: "shop"
        items: "1,2,3,4,5"    # 填物品模板 ID，逗号分隔
```

### 修改物品价格

在 `ItemSeeder.kt` 中修改对应物品的 `price` 字段。

### 修改卖出比例

当前卖出价 = 买入价 / 2（硬编码在代码中）。如果需要修改，搜索 `template.price / 2`。

### 修改初始金币

在 `attributes.yaml` 中修改 `gold` 的 `defaultValue`：

```yaml
  gold:
    tier: 1
    displayName: "金币"
    defaultValue: 100    # 新角色初始 100 金（原来是 50）
```

## 14. 系统是怎么运作的（简要说明）

### 整体流程

```
玩家在 QQ / 控制台输入指令
  ↓
指令路由器（CommandRouter）匹配指令
  ↓
检查前置条件（等级够吗？在对的地方吗？）
  ↓
扣除消耗（MP、行动力等）
  ↓
执行处理器（Handler）
  ↓
返回结果文字给玩家
```

### 核心引擎

| 引擎 | 管什么 | 类比 |
|------|--------|------|
| 属性引擎 | 所有数值（HP/MP/攻击/防御…） | Excel 公式表 |
| 特效引擎 | 所有"动作"（伤害/治疗/加Buff…） | 动作指令集 |
| 技能引擎 | 组合特效成技能 | 把多个动作打包 |
| Buff 系统 | 持续性效果（灼烧/护盾…） | 挂在角色身上的定时器 |
| 战斗系统 | 回合制战斗循环 | 自动裁判 |
| 地图系统 | 区域连接和移动 | 棋盘 |
| 任务系统 | 目标追踪和奖励 | 任务清单 |
| NPC 系统 | 角色和对话 | 演员列表 |

### 配置怎么变成游戏的

```
YAML 配置文件
  ↓ 启动时加载
内存中的数据结构
  ↓ 玩家操作时
引擎读取数据，执行逻辑
  ↓
结果存入 SQLite 数据库（角色存档）
```

**记住：** 改配置 = 改游戏内容。改代码 = 改游戏规则。大部分时候你只需要改配置。

## 15. 常见错误排查

### 改完配置后游戏启动报错

**最常见原因：YAML 格式错误。**

检查清单：
- [ ] 冒号后面有空格吗？（`name: "值"` 不是 `name:"值"`）
- [ ] 缩进用的是空格不是 Tab 吗？
- [ ] 引号是英文引号吗？
- [ ] 列表的 `-` 后面有空格吗？

### 新加的敌人/NPC 不出现

- 敌人：需要在 `nodes.yaml` 的对应节点的 `entities.enemies` 里加上 key
- NPC：需要设置正确的 `location`（对应 nodes.yaml 的节点 key）

### 新加的物品 ID 对不上

添加新物品后，需要删除 `master.sqlite` 重启，让数据库重建。物品 ID 按代码中的顺序从 1 递增。

### 技能在战斗中不显示

- 检查技能 key 是否加到了敌人的 `skills` 列表里（敌人用）
- 玩家默认拥有 skills.yaml 中的所有技能

### NPC 对话只显示"没有回应"

检查 `app.yaml` 中的 AI 配置（apiKey 是否正确，网络是否通畅）。AI 对话是可选功能，不影响其他游戏功能。
