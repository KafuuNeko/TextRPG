## 1. 你需要准备什么

| 东西 | 用途 | 必须？ |
|------|------|--------|
| 一台电脑 | 运行游戏服务器（Windows / Mac / Linux 均可） | 是 |
| Java 17 | 游戏运行环境 | 是 |
| QQ 机器人框架 | 让游戏能收发 QQ 消息（推荐 NapCat） | 是 |

## 2. 第一步：安装 Java

游戏需要 **Java 17** 或更高版本。

### Windows

1. 打开浏览器，访问 https://adoptium.net/
2. 点击 **Latest LTS Release** 下载（选 `.msi` 安装包）
3. 双击安装，一路点 **Next**
4. 安装时勾选 **"Set JAVA_HOME variable"**（重要！）

### 验证安装

打开命令提示符（按 `Win + R`，输入 `cmd`，回车），输入：

```
java -version
```

看到类似下面的输出就对了：

```
openjdk version "17.0.x" ...
```

如果提示"不是内部或外部命令"，说明安装没成功或环境变量没配好，重新安装一次试试。

## 4. 第三步：安装 QQ 机器人框架

游戏通过 **OneBot 协议** 和 QQ 通信。你需要一个 OneBot 实现来做中间桥梁。

### 推荐：NapCat

NapCat 是目前最简单的选择。

1. 访问 https://github.com/NapNeko/NapCatQQ/releases
2. 下载最新版本（Windows 选 `.exe` 或 `.zip`）
3. 解压 / 安装
4. 启动 NapCat，用你的 **QQ 小号** 扫码登录

### 配置 NapCat

登录成功后，打开 NapCat 的网页配置面板（通常是 `http://127.0.0.1:6099`）：

1. 找到 **网络配置** 或 **连接设置**
2. 添加一个 **反向 WebSocket** 连接：
   - 类型：`反向 WebSocket`
   - 地址：`ws://127.0.0.1:8080`（这是游戏默认监听的地址）

> **注意**：NapCat 的版本和界面可能有差异，核心是确保它通过 WebSocket 连接到 `ws://127.0.0.1:8080`。如果 NapCat 只支持正向 WebSocket，参考其文档配置。

### 其他可选框架

| 框架 | 说明 |
|------|------|
| LLOneBot | 另一个 OneBot 实现，配置类似 |
| Lagrange | 轻量级方案 |
| go-cqhttp | 老牌方案（已停更，但仍可用） |

只要支持 OneBot v11 协议 + WebSocket 连接即可。

## 5. 第四步：修改配置

打开项目中的配置文件：

```
src/main/resources/config/app.yaml
```

用任意文本编辑器（记事本就行）打开：

```yaml
bot:
  enabled: true
  websocketUrl: "ws://127.0.0.1:8080"   # QQ 机器人的 WebSocket 地址
  accessToken: ""                         # 如果设置了 token 就填，没有就留空

llm:
  enabled: true
  apiUrl: "https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions"
  apiKey: "你的API密钥"                   # 千问 API Key（可在 https://dashscope.console.aliyun.com/ 申请）
  model: "qwen-plus"
  timeout: 30000

database:
  url: "jdbc:sqlite:master.sqlite"        # 数据库文件路径，不用改
  driver: "org.sqlite.JDBC"
```

### 需要改什么？

- **websocketUrl**：如果你的 QQ 机器人框架用了不同的端口，改这里
- **apiKey**：换成你自己的千问 API Key
  - 没有 API Key？NPC 对话和探索功能会显示占位文本，其他功能正常使用
- 其他选项一般不需要改

## 6. 第五步：启动游戏

### Windows

在项目根目录，打开命令提示符（在文件夹空白处按住 Shift + 右键 → "在此处打开命令窗口"），输入：

**双击项目根目录的 `start.bat`** 即可启动。

它会自动处理中文编码问题并启动游戏。第一次运行需要下载依赖，可能等几分钟。

> 如果你想手动启动或遇到问题，也可以打开 cmd，依次输入：
> ```
> chcp 65001
> gradlew.bat run --console=plain
> ```
> `chcp 65001` 是把 cmd 切换成 UTF-8 编码，不然中文会乱码。

### Mac / Linux

```bash
./gradlew run
```

> 如果提示权限不足，先执行 `chmod +x gradlew`。Mac / Linux 不存在编码问题。

### 两种运行模式

游戏支持两种模式，通过 `app.yaml` 中的 `bot.enabled` 切换：

| 模式 | bot.enabled | 说明 |
|------|-------------|------|
| **控制台模式** | `false` | 直接在终端输入指令，无需 QQ 机器人，适合本地测试 |
| **QQ 机器人模式** | `true` | 连接 OneBot 框架，通过 QQ 收发消息 |

**默认是控制台模式**，不需要任何额外配置就能直接玩！

### 控制台模式启动成功的样子

```
━━━━━━━━━━━━━━━━━━━━━━━━
  TextRPG 控制台模式
━━━━━━━━━━━━━━━━━━━━━━━━
直接输入指令即可游玩（如 /注册 勇者）
输入 quit 退出

>
```

看到 `>` 提示符就可以开始输入指令了！比如：

```
> /注册 勇者
━━━ 角色创建成功 ━━━
名称：勇者
等级：Lv.1
位置：村庄广场
获得：木剑 x1、治疗药水 x3

> /状态
━━━━━ 角色状态 ━━━━━
等级：Lv.1  经验：0
金币：50
位置：村庄广场
...
```

输入 `quit` 保存数据并退出。

### QQ 机器人模式启动成功的样子

将 `app.yaml` 中 `bot.enabled` 改为 `true` 后：

```
TextRPG 启动完成，等待 OneBot 连接...
```

然后在 QQ 里对机器人发消息即可。

| 指令 | 说明 | 示例 |

| `/注册 <名字>` | 创建角色 | `/注册 勇者小明` |

| `/移动 <地名>` | 移动到相邻区域 | `/移动 村口` |

| 指令 | 说明 | 示例 |

| `/战斗 <敌人>` | 和当前区域的敌人战斗 | `/战斗 史莱姆` |

   → 获得 50 经验 + 100 金币 + 3 治疗药水

8. /商店 买 铁剑             ← 花 150 金买更好的武器

## 9. 常见问题

### Q: 启动后报 "Connection failed"

机器人框架还没启动，或者 WebSocket 地址不对。
- 确认 NapCat 已经启动并登录
- 确认 `app.yaml` 里的 `websocketUrl` 和 NapCat 配置一致

### Q: 发指令没反应

- 确认消息前面有 `/`（这是指令前缀）
- 确认机器人在群里（或私聊机器人）
- 看游戏控制台有没有 `收到消息` 的日志

### Q: "你已经在战斗中了"

上次战斗可能因为超时卡住了。重启游戏服务即可。

### Q: NPC 对话显示"没有回应"

- 检查 `app.yaml` 里的 `apiKey` 是否正确
- 检查网络能否访问 `dashscope.aliyuncs.com`
- AI 功能是可选的，不影响其他游戏功能

### Q: 想换一个 AI 模型

在 `app.yaml` 中修改 `llm` 部分。任何兼容 OpenAI Chat Completions API 的服务都可以：

```yaml
# 本地 Ollama
llm:
  apiUrl: "http://127.0.0.1:11434/v1/chat/completions"
  apiKey: ""
  model: "qwen2.5"

# OpenAI
llm:
  apiUrl: "https://api.openai.com/v1/chat/completions"
  apiKey: "sk-xxx"
  model: "gpt-4o-mini"

# 其他兼容服务（DeepSeek、智谱等）
# 只要是 OpenAI 格式的 API 都行
```

### Q: 想重置游戏数据

删除项目根目录下的 `master.sqlite` 文件，重新启动即可。所有角色数据会被清空。

### Q: 第一次运行下载很慢

Gradle 在下载依赖。如果太慢，可以配置国内镜像。在项目根目录创建或编辑 `gradle.properties`，添加：

```properties
systemProp.https.proxyHost=你的代理地址
systemProp.https.proxyPort=你的代理端口
```

或者使用阿里云 Gradle 镜像，在 `build.gradle.kts` 的 `repositories` 中添加：

```kotlin
repositories {
    maven { url = uri("https://maven.aliyun.com/repository/public") }
    mavenCentral()
}
```
